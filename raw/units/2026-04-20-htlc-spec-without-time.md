## Source Code

Check Hash Time Lock Contracts without discrete time as a state variable.

source file

```tlaplus
---- MODULE htlc ----
CONSTANTS Alice, Bob, Contract
VARIABLES owner, locked, revealed, refunded

vars == <<owner, locked, revealed, refunded>>

Init ==
	/\ owner = Alice
	/\ locked = FALSE
	/\ revealed = FALSE
	/\ refunded = FALSE

Lock ==
	/\ ~locked /\ ~revealed /\ ~refunded
	/\ owner = Alice
	/\ owner' = Contract
	/\ locked' = TRUE
	/\ UNCHANGED <<revealed, refunded>>

BobReveal ==
	/\ locked
	/\ ~revealed /\ ~refunded
	/\ owner' = Bob
	/\ locked' = FALSE
	/\ revealed' = TRUE
	/\ UNCHANGED refunded

AliceRefund ==
	/\ locked
	/\ ~revealed /\ ~refunded
	/\ owner' = Alice
	/\ locked' = FALSE
	/\ refunded' = TRUE
	/\ UNCHANGED revealed

Next == 
	\/ Lock
	\/ BobReveal
	\/ AliceRefund

Spec ==
	/\ Init
	/\ [][Next]_vars
	/\ WF_vars(BobReveal)
	/\ WF_vars(AliceRefund)

Atomicity == ~(revealed /\ refunded)

Conservation == owner \in {Alice, Bob, Contract}

Custody ==
	/\ (locked => owner = Contract)
	/\ ((revealed \/ refunded) => owner \in {Alice, Bob})
	/\ (revealed => owner = Bob)
	/\ (refunded => owner = Alice)

Terminality == (revealed \/ refunded) => ~locked

Progress == locked ~> (revealed \/ refunded)

====
```

cfg file:

```
CONSTANTS
	Alice = a
	Bob = b
	Contract = c

SPECIFICATION Spec

INVARIANTS
	Atomicity
	Conservation
	Custody
	Terminality

PROPERTY Progress

CHECK_DEADLOCK FALSE
```

## Debug Traces

### First attempt — records

Packing all variables into a single record does not work as expected. I failed to notice that `revealed` and `refund` are **mutually exclusive** state variables.

> [!TIP]
> Mutually exclusive state variables can be modeled as a _sum type_ or represented using boolean signals.

Here, boolean signals are used because the initial state must be neither `revealed` nor `refund`.

## Units

1. **Declare participants and asset holder as symbolic `CONSTANTS`** — lift the parties (including the escrow role itself) into `CONSTANTS` so the spec is parameterized over identities and the model gets concrete values only in the cfg. ♻️ reusable
    
2. **Model ownership transfer via a single `owner` variable over a constant set** — encode "who holds the asset" as one variable ranging over the participant constants, instead of per-party boolean flags, so transfers are single-assignment and conservation is expressible as set membership. ♻️ reusable
    
3. **Use boolean phase flags for mutually exclusive outcomes** — track protocol phases (`locked`, `revealed`, `refunded`) as separate booleans so preconditions can combine them with negation, and terminal-state disjointness becomes a propositional invariant. ♻️ reusable
    
4. **Bundle state into a `vars` tuple for stutter-step framing** — define `vars == <<...>>` once and reuse it in `[Next]_vars` and `WF_vars(...)` so the framing subscript stays in sync with the actual state. ♻️ reusable
    
5. **Guard actions with negated phase flags as preconditions** — prefix each action with `~locked /\ ~revealed /\ ~refunded` (or the subset that applies) so actions self-gate on protocol phase without a separate `pc` variable. ♻️ reusable
    
6. **Pair every primed update with explicit `UNCHANGED` for the rest** — every action names exactly the variables it changes and closes with `UNCHANGED <<...>>` on the remainder, so no variable's next-state is accidentally unconstrained. ♻️ reusable
    
7. **Express the transition relation as a disjunction of named actions** — define `Next == \/ A1 \/ A2 \/ ...` so each action is an independently reviewable operator and the top-level step is a menu of choices. ♻️ reusable
    
8. **Assemble `Spec` as `Init /\ [][Next]_vars /\ fairness`** — canonical TLA+ spec shape: initial predicate, stuttering box-next, then fairness conjuncts, so liveness assumptions live in the spec, not the property. ♻️ reusable
    
9. **Attach `WF_vars` only to actions that must eventually fire** — add weak fairness per-action (here `BobReveal` and `AliceRefund`) rather than a blanket fairness on `Next`, so you commit to _which_ branches are guaranteed to be scheduled. ♻️ reusable
    
10. **State mutual exclusion of terminal outcomes as a negated-conjunction invariant** — write `Atomicity == ~(revealed /\ refunded)` to forbid both end states holding at once; a propositional safety property needs no temporal operator. ♻️ reusable
    
11. **Type-constrain a variable via set-membership invariant** — write `owner \in {Alice, Bob, Contract}` as an invariant instead of relying on declared types, so the model checker enforces the domain at every state. ♻️ reusable
    
12. **Encode per-phase ownership with implication chains** — conjoin `phase => owner = party` implications (one per phase) as a single invariant so each phase's custody obligation is independently readable. ♻️ reusable
    
13. **Express "terminal states exclude an intermediate flag" as an implication** — `(revealed \/ refunded) => ~locked` states a cross-phase invariant as a one-directional implication, keeping it distinct from equivalence. ♻️ reusable
    
14. **State liveness with the leads-to operator `~>`** — write `locked ~> (revealed \/ refunded)` so progress is a temporal property, not an invariant; pairs with the `WF_vars` fairness conjuncts that make it provable. ♻️ reusable
    
15. **Split checker obligations between `INVARIANTS` and `PROPERTY` in the cfg** — route state predicates to `INVARIANTS` and temporal formulas to `PROPERTY` so TLC checks each with the appropriate engine. ♻️ reusable
    
16. **Bind model-level constants to short symbolic values in the cfg** — `Alice = a`, `Bob = b`, `Contract = c` assigns concrete model values to the spec's `CONSTANTS` without polluting the spec with them. ♻️ reusable
    
17. **Point the cfg at the assembled `Spec` rather than `Init`/`Next` separately** — use `SPECIFICATION Spec` so fairness conjuncts travel with the spec instead of being re-declared in the cfg. ♻️ reusable
    
18. **Set `CHECK_DEADLOCK FALSE` when terminal states are legitimate** — disable deadlock checking for protocols whose success is reaching a fixed endpoint, so TLC doesn't flag the intended terminal as a violation. 📍 context-bound
    
19. **Model time-locked contracts without a clock variable** — drop discrete time entirely and rely on non-determinism between competing actions (`BobReveal` vs `AliceRefund`) plus fairness, so the spec checks the _outcome_ structure without enumerating timestamps. ♻️ reusable
